# Le Fusil de Tchekhov (Chekhov's Gun)

## Principe
"Si vous dites dans le premier chapitre qu'il y a un fusil accroché au mur, alors il faut absolument qu'un coup de feu parte avec au second ou au troisième chapitre. S'il n'est pas destiné à être utilisé, il n'a rien à faire là." — Anton Tchekhov.

## Application : La Causalité
Tout élément introduit dans l'histoire est une promesse faite au lecteur.
- **Promesse** : "Il a une cicatrice étrange." -> **Paiement** : On doit apprendre l'origine de cette cicatrice.
- **Promesse** : "Il déteste les chiens." -> **Paiement** : Il devra interagir avec un chien à un moment critique.

## Pour l'IA
L'IA a tendance à ajouter des détails "décoratifs" qu'elle oublie aussitôt.
**Règle** : Chaque "Objet" ou "Trait" significatif doit être loggé dans la base de données (Context DB) pour être réutilisé plus tard.

## Nuance : Le "Red Herring"
On peut montrer un fusil qui ne tire pas, *si* c'est pour tromper volontairement le lecteur (fausse piste). Mais cela doit être un choix délibéré, pas un oubli.
